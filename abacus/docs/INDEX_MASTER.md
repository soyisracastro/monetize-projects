## 🔄 PROCESO DE ACTUALIZACIÓN

### Cuándo actualizar documentación

```
✅ Cada vez que lanzas una nueva fase
✅ Cuando cambias infraestructura
✅ Cuando pivotas estrategia
✅ Al final de cada mes (review)
✅ Cuando encuentras errores
```

### Versionamiento

```
Formato: X.Y

X = Cambio mayor (nueva fase, pivot)
Y = Cambio menor (correcciones, adiciones)

Ejemplos:
1.0 → MVP inicial
1.1 → Correcciones MVP
2.0 → Fase 2 lanzada
2.1 → Ajustes Fase 2
```

### Changelog

Cada documento debe tener sección de changelog:

```markdown
## CHANGELOG

### v1.1 (15 Marzo 2026)
- Agregada sección de troubleshooting
- Actualizado pricing
- Corregidos typos

### v1.0 (11 Marzo 2026)
- Versión inicial
```
